# MQTT Viewer - User Guide

## Overview
MQTT Viewer is a desktop application for viewing, publishing, and managing MQTT messages in real-time. Similar to MQTTX, it provides a clean interface to interact with MQTT brokers.

## Getting Started

### 1. Connection Tab 🔌
First, you need to connect to an MQTT broker.

**Connection Settings:**
- **Broker URL**: The address of your MQTT broker (e.g., `mqtt://localhost:1883`)
- **Client ID**: Unique identifier for this connection (auto-generated by default)
- **Username/Password**: Authentication credentials (optional)
- **Keepalive**: Time interval for keeping the connection alive (seconds)
- **Reconnect Period**: Time to wait before attempting to reconnect (milliseconds)

**Save Connections:**
Click "Save Connection" to store your broker settings for quick access later.

### 2. Subscribe Tab 📨
Subscribe to MQTT topics to receive messages in real-time.

**Topic Wildcards:**
- `+` - Matches a single level in the topic hierarchy
  - Example: `sensor/+/temperature` matches `sensor/room1/temperature` and `sensor/room2/temperature`
- `#` - Matches multiple levels (only at the end)
  - Example: `sensor/#` matches `sensor/room1/temperature`, `sensor/room1/humidity`, etc.

**QoS Levels:**
- **0 - At most once**: Fire and forget
- **1 - At least once**: Message guaranteed to arrive
- **2 - Exactly once**: Highest reliability

**Active Subscriptions:**
View all active subscriptions with message counts. Unsubscribe by clicking the "Unsubscribe" button.

### 3. Publish Tab 📤
Send messages to MQTT topics.

**Options:**
- **Topic**: The target topic
- **Payload**: Your message (supports JSON, plain text, etc.)
- **QoS**: Quality of Service level
- **Retain**: If enabled, the message is retained by the broker

**Templates:**
Use quick templates for common message types (JSON, Plain Text, Numeric).

### 4. Messages Tab 💬
View all received messages in real-time.

**Features:**
- **Real-time Updates**: Messages appear as they arrive
- **Filtering**: Filter by topic name or message content
- **Auto-scroll**: Automatically scroll to the latest message
- **QoS & Retain Badges**: Visual indicators for each message
- **Copy to Clipboard**: Click the copy button to copy message content
- **Statistics**: See total message count and subscription count

## Tips & Tricks

1. **Use Wildcards Effectively**: Subscribe to `#` to see all messages (careful in production!)
2. **Save Frequently Used Connections**: Avoid re-entering credentials
3. **Check QoS Levels**: Adjust QoS based on message importance
4. **Use Retain for State**: Publish state messages with retain=true for persistence
5. **Filter Messages**: Use the filter to find specific topics or content
6. **Clear Messages**: Clear the message history to reduce memory usage

## Common Use Cases

### Monitor Temperature Sensor
1. Subscribe to: `home/sensors/+/temperature`
2. Watch messages arrive from all rooms
3. Publish test data: `home/sensors/room1/temperature` with payload `22.5`

### Save Connection State
1. Connect to your broker
2. Publish: `home/state/light` with message `on` and Retain=true
3. This state persists even after disconnect/reconnect

### Debugging MQTT Topics
1. Subscribe to `#` to see all messages
2. Look at the Messages tab to understand topic structure
3. Use filters to focus on specific areas

## Keyboard Shortcuts
- **Enter** in topic field: Subscribe
- **Ctrl+L**: Focus on filter input

## Troubleshooting

**Cannot Connect to Broker**
- Verify broker URL and port
- Check firewall settings
- Ensure broker is running

**Messages Not Appearing**
- Confirm you're subscribed to the correct topic
- Check QoS settings
- Verify broker connectivity status

**Connection Keeps Dropping**
- Increase Keepalive value
- Check network stability
- Review broker logs

## Architecture

- **mqttStore.js**: Pinia store managing all MQTT state
- **Mqtt.js**: MQTT client wrapper with Promise support
- **Components**: Vue components for each tab
- **MqttViewerView.vue**: Main container with tabbed interface
